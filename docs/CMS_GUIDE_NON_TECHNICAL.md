## 1. Membuat Akun GitHub / Creating a GitHub Account

Untuk mengakses CMS, Anda perlu akun GitHub. Ikuti langkah berikut:
To access the CMS, you need a GitHub account. Follow these steps:

### Langkah 1 — Buka halaman pendaftaran / Open the signup page

1. Buka browser dan kunjungi **github.com** (Open your browser and go to **github.com**)
2. Klik tombol **Sign up** di pojok kanan atas (Click the **Sign up** button in the top right corner)

### Langkah 2 — Isi formulir pendaftaran / Fill out the signup form

3. Masukkan **alamat email** Anda (Enter your **email address**)
4. Buat **password** yang kuat — minimal 8 karakter, campuran huruf dan angka (Create a strong **password** — at least 8 characters, mix of letters and numbers)
5. Pilih **username** — ini akan menjadi identitas Anda di GitHub (Choose a **username** — this will be your identity on GitHub)
6. Klik **Create account** (Click **Create account**)

### Langkah 3 — Verifikasi email / Verify your email

7. GitHub akan mengirim email verifikasi ke alamat email yang Anda daftarkan (GitHub will send a verification email to the address you signed up with)
8. Buka email tersebut dan klik **Verify email address** (Open that email and click **Verify email address**)
9. Selesaikan pertanyaan verifikasi di halaman GitHub jika diminta (Complete any verification questions on the GitHub page if prompted)

### Langkah 4 — Minta akses dari admin / Request access from admin

10. Hubungi **admin tim** (koordinator IT atau pengembang) dan berikan username GitHub Anda (Contact the **team admin** — the IT coordinator or developer — and give them your GitHub username)
11. Admin akan menambahkan Anda sebagai **collaborator** ke repository proyek (The admin will add you as a **collaborator** to the project repository)

### Langkah 5 — Terima undangan / Accept the invitation

12. Anda akan menerima email undangan dari GitHub (You will receive an invitation email from GitHub)
13. Klik tombol **Accept invitation** di email tersebut (Click the **Accept invitation** button in the email)
14. Sekarang Anda sudah memiliki akses! (You now have access!)

## 2. Membuka CMS / Opening the CMS

### Langkah 1 — Buka halaman CMS / Open the CMS page

1. Buka browser dan kunjungi:
   (Open your browser and go to:)

   **https://melukis-sambelia.vercel.app/admin**

### Langkah 2 — Login dengan GitHub / Log in with GitHub

2. Klik tombol **Login with GitHub** (Click the **Login with GitHub** button)

### Langkah 3 — Otorisasi aplikasi / Authorize the app

3. GitHub akan meminta izin untuk menghubungkan akun Anda dengan CMS. Klik **Authorize** (GitHub will ask for permission to connect your account to the CMS. Click **Authorize**)
4. Jika ini pertama kali, Anda mungkin perlu mengonfirmasi ulang dengan password GitHub (If this is your first time, you may need to re-confirm with your GitHub password)

### Langkah 4 — Anda sudah masuk! / You're in!

5. Anda akan melihat dashboard CMS dengan sidebar di sebelah kiri (You will see the CMS dashboard with a sidebar on the left)

## 3. Navigasi CMS / CMS Navigation

Setelah masuk, Anda akan melihat tata letak berikut:
After logging in, you will see the following layout:

### Sidebar (bilah sisi kiri / left sidebar)

Sidebar menampilkan semua **koleksi** (collections) — setiap koleksi mewakili satu jenis konten di website:
The sidebar shows all **collections** — each collection represents one type of content on the website:

| Koleksi / Collection | Isi / Content |
|---|---|
| **Pariwisata** | Destinasi wisata / Tourism destinations |
| **Irigasi** | Data saluran irigasi / Irrigation data |
| **Kesehatan** | Fasilitas kesehatan / Health facilities |
| **UMKM** | Usaha mikro kecil menengah / Local businesses |
| **Festival** | Acara festival / Festival events |
| **Kegiatan** | Artikel kegiatan / Activity articles |

### Cara browsing konten / How to browse content

1. Klik nama koleksi di sidebar untuk melihat daftar semua item (Click a collection name in the sidebar to see all items)
2. Klik item mana pun untuk membuka dan mengedit (Click any item to open and edit it)

### Penjelasan Field Umum / Common Field Explanations

#### Apa itu "Urutan" (order)?

- Field **Urutan** (order) menentukan urutan tampilan item di website.
- **Angka lebih kecil tampil lebih dulu.** Contoh: item dengan `order: 1` muncul sebelum item dengan `order: 5`.
- Jika dikosongkan, item akan diurutkan berdasarkan tanggal pembuatan.
- The **order** field determines the display order of items on the website.
- **Smaller numbers appear first.** E.g., an item with `order: 1` appears before `order: 5`.
- If left empty, items are sorted by creation date.

#### Apa itu "Link Google Maps" (googleMapsUrl)?

- Jika koordinat latitude/longitude tidak menunjukkan lokasi yang tepat di peta, Anda bisa menambahkan link Google Maps langsung ke lokasi yang benar.
- **Cara mendapatkan link:** Buka Google Maps → cari lokasi → klik "Bagikan" (Share) → salin link.
- If the latitude/longitude coordinates don't point to the exact location on the map, you can add a direct Google Maps link to the correct spot.
- **How to get the link:** Open Google Maps → find the location → click "Share" → copy the link.

#### Apa itu field "Konten" (body)?

- Field **Konten** (body) menggunakan format **Markdown** untuk menulis teks.
- Anda bisa menggunakan format berikut:
  - `**teks tebal**` → **teks tebal** (bold text)
  - `*teks miring*` → *teks miring* (italic text)
  - `- item` → daftar bullet (bullet list)
  - `# Judul` → heading besar (large heading)
- The **body** field uses **Markdown** formatting.
  - `**bold text**` → **bold text**
  - `*italic text*` → *italic text*
  - `- item` → bullet list
  - `# Heading` → large heading

### Cara Menemukan Koordinat Latitude & Longitude / How to Find Latitude & Longitude Coordinates

1. Buka **https://www.openstreetmap.org** di browser (Open **https://www.openstreetmap.org** in your browser)
2. Cari lokasi yang Anda inginkan di peta (Search for the location you want on the map)
3. **Klik kanan** pada lokasi yang tepat di peta (**Right-click** on the exact location on the map)
4. Pilih **"Show address"** atau perhatikan koordinat yang muncul — formatnya adalah `latitude, longitude` (Select **"Show address"** or note the coordinates that appear — the format is `latitude, longitude`)
5. Contoh: `-8.3583, 116.8417` berarti **Latitude = -8.3583** dan **Longitude = 116.8417** (Example: `-8.3583, 116.8417` means **Latitude = -8.3583** and **Longitude = 116.8417**)
6. Masukkan angka tersebut ke field **Latitude** dan **Longitude** di CMS (Enter those numbers into the **Latitude** and **Longitude** fields in the CMS)

> **Alternatif:** Di Google Maps, klik kanan pada lokasi → koordinat muncul di bagian atas. Salin dan pisahkan menjadi latitude dan longitude.
>
> **Alternative:** In Google Maps, right-click on a location → coordinates appear at the top. Copy and separate into latitude and longitude.

### Cara Upload Gambar / How to Upload Images

1. Klik field **gambar** (misalnya "Cover" atau "Galeri") di form CMS (Click the **image** field — e.g., "Cover" or "Galeri" — in the CMS form)
2. Klik tombol **Choose** (Click the **Choose** button)
3. Pilih salah satu:
   - **Upload dari komputer:** Klik "Upload" → pilih file dari komputer Anda (Upload from your computer)
   - **Pilih dari Media Library:** Pilih gambar yang sudah diupload sebelumnya (Choose from previously uploaded images)
4. Setelah gambar dipilih, path akan terisi otomatis (After selecting, the path will be filled automatically)

**Tips gambar / Image tips:**

- Format terbaik: **JPG** untuk foto, **PNG** untuk logo, **WebP** untuk ukuran terkecil
- Best format: **JPG** for photos, **PNG** for logos, **WebP** for smallest file size
- Rasio (aspect ratio):
  - Cover Pariwisata/UMKM/Kegiatan: **16:9** (landscape)
  - Gambar Desa: **16:9** (landscape)
- Ukuran maksimal: **< 500KB** per gambar. Jika terlalu besar, kompres di **https://squoosh.app** atau **https://tinypng.com**
- Maximum size: **< 500KB** per image. If too large, compress at **https://squoosh.app** or **https://tinypng.com**
- Nama file: gunakan huruf kecil tanpa spasi. Contoh: `pantai-berandangan.webp` (bukan `Pantai Berandangan.jpg`)
- File name: use lowercase without spaces. Example: `pantai-berandangan.webp` (not `Pantai Berandangan.jpg`)

## 5. Mengedit Konten / Editing Existing Content

### Langkah-langkah / Steps

1. Di sidebar CMS, klik koleksi yang berisi konten yang ingin diedit (In the CMS sidebar, click the collection containing the content you want to edit)
   - Contoh: klik **Pariwisata** untuk mengedit data wisata (Example: click **Pariwisata** to edit tourism data)

2. Klik item yang ingin diedit dari daftar (Click the item you want to edit from the list)

3. Ubah field yang diinginkan (Modify the desired fields)
   - Klik di dalam field dan edit teks (Click inside a field and edit the text)
   - Untuk gambar: klik gambar → pilih gambar baru atau hapus (For images: click the image → choose a new one or delete)
   - Untuk daftar (list): klik "Add" untuk menambah, atau "×" untuk menghapus item (For lists: click "Add" to add, or "×" to remove an item)

4. Klik tombol **Save** di pojok kanan atas untuk menyimpan tanpa mempublikasikan, ATAU klik **Publish** untuk menyimpan dan langsung mempublikasikan (Click **Save** in the top right to save without publishing, OR click **Publish** to save and publish immediately)

5. Jika Anda menyimpan sebagai draft (Save saja), item akan bertanda "Draft". Untuk mempublikasikannya nanti, buka kembali item tersebut lalu klik **Publish** (If you save as a draft, the item will show a "Draft" badge. To publish it later, open the item again and click **Publish**)

### Penting untuk Diingat / Important to Remember

- **Selalu klik Save atau Publish** sebelum meninggalkan halaman, jika tidak perubahan akan hilang! (Always click Save or Publish before leaving the page, or your changes will be lost!)
- Setelah klik **Publish**, perubahan membutuhkan **1-2 menit** untuk muncul di website (After clicking **Publish**, changes take **1-2 minutes** to appear on the website)

Bagian **Pengaturan Situs** (Site Settings) mengatur tampilan global website — dari gambar hero hingga teks footer dan pesan kosong.
The **Site Settings** section controls the global website appearance — from the hero image to footer text and empty state messages.

Di sidebar CMS, klik **Pengaturan Situs** → **Pengaturan** untuk membuka form pengaturan.
In the CMS sidebar, click **Pengaturan Situs** → **Pengaturan** to open the settings form.

## 7. Tips & Troubleshooting / Tips & Penanganan Masalah

### Tips

1. **Perubahan butuh waktu.** Setelah klik Publish, perubahan membutuhkan **1-2 menit** untuk muncul di website. Ini normal — website sedang dibangun ulang.
   **Changes take time.** After clicking Publish, changes take **1-2 minutes** to appear on the website. This is normal — the website is being rebuilt.

2. **Simpan secara berkala.** Jika Anda mengedit konten yang panjang, klik **Save** secara berkala agar tidak kehilangan pekerjaan jika browser tertutup.
   **Save periodically.** If editing long content, click **Save** periodically to avoid losing work if the browser closes.

3. **Gunakan format Markdown untuk konten.** Di field "Konten" (body), gunakan format markdown untuk membuat teks tebal, miring, daftar, dll.
   **Use Markdown for content.** In the "body" field, use Markdown formatting for bold, italic, lists, etc.

4. **Kompres gambar sebelum upload.** Gunakan https://squoosh.app atau https://tinypng.com untuk mengecilkan ukuran file gambar sebelum upload. Idealnya di bawah 500KB.
   **Compress images before uploading.** Use https://squoosh.app or https://tinypng.com to reduce image file size before uploading. Ideally under 500KB.

5. **Nama file gambar.** Gunakan huruf kecil tanpa spasi. Contoh baik: `pantai-berandangan.webp`. Contoh buruk: `Pantai Berandangan.JPG`.
   **Image file names.** Use lowercase without spaces. Good example: `pantai-berandangan.webp`. Bad example: `Pantai Berandangan.JPG`.

6. **Jangan hapus field yang tidak dipahami.** Jika tidak yakin dengan suatu field, lebih baik ditanyakan ke tim teknis daripada dihapus.
   **Don't delete fields you don't understand.** If unsure about a field, ask the technical team rather than deleting it.

7. **Urutan (order) itu penting.** Ingat: angka lebih kecil = tampil lebih dulu. Jika ingin item baru tampil di atas, beri angka kecil (misalnya 1 atau 0).
   **Order matters.** Remember: smaller numbers = appear first. If you want a new item at the top, give it a small number (e.g., 1 or 0).

### Troubleshooting / Penanganan Masalah

| Masalah / Problem | Solusi / Solution |
|---|---|
| **Tidak bisa login ke CMS** / Cannot log in to CMS | Pastikan Anda sudah menerima dan menerima undangan collaborator dari GitHub. Cek email untuk undangan. / Make sure you have received and accepted the collaborator invitation from GitHub. Check your email for the invitation. |
| **Perubahan tidak muncul di website** / Changes don't appear on website | Tunggu 1-2 menit. Jika masih tidak muncul, cek apakah Anda klik **Publish** (bukan hanya **Save**). / Wait 1-2 minutes. If still not appearing, check if you clicked **Publish** (not just **Save**). |
| **Gambar tidak muncul** / Image doesn't appear | Pastikan gambar berhasil diupload (bukan hanya path teks). Hapus dan upload ulang gambar. / Make sure the image was actually uploaded (not just a text path). Delete and re-upload the image. |
| **Koordinat salah di peta** / Wrong coordinates on map | Periksa kembali angka latitude dan longitude. Latitude untuk Lombok sekitar **-8.3** (negatif!), longitude sekitar **116.8**. / Double-check the latitude and longitude numbers. Latitude for Lombok is approximately **-8.3** (negative!), longitude approximately **116.8**. |
| **Field wajib tidak terisi** / Required field not filled | CMS tidak akan menyimpan jika field wajib (ditandai *) kosong. Isi semua field wajib terlebih dahulu. / The CMS won't save if required fields (marked with *) are empty. Fill all required fields first. |
| **Tidak bisa menambah item baru** / Cannot add new item | Pastikan Anda berada di koleksi yang benar dan klik tombol **New** di pojok kanan atas. / Make sure you're in the correct collection and click the **New** button in the top right. |
| **Browser menampilkan versi lama** / Browser shows old version | Tekan **Ctrl+Shift+R** (Windows) atau **Cmd+Shift+R** (Mac) untuk refresh paksa. / Press **Ctrl+Shift+R** (Windows) or **Cmd+Shift+R** (Mac) for a hard refresh. |
| **Lupa password GitHub** / Forgot GitHub password | Kunjungi github.com → klik "Forgot password?" → ikuti instruksi reset. / Visit github.com → click "Forgot password?" → follow the reset instructions. |
